Revision and Reflection

My webpage’s strongest feature is the overall design and theme. I think the dark colors, pink accents, and futuristic fonts fit the Y2K aesthetic really well and make the site visually interesting. The layout is also organized clearly with sections, cards, and grids that make the information easy to follow. One issue I noticed was that my images did not include alt text, which could make the website less accessible. Another issue was that my navigation links were not connected to actual sections of the page yet. To fix these problems, I would add alt text to the images and connect the navigation menu to the different sections so the site functions better and is easier to use.

Progressive Enhancement

Progressive enhancement is a web design approach where you focus on the main content first, then build everything else around it. The idea is to make sure the website works for everyone, regardless of their browser, device, or internet connection. First, you create the basic structure with simple HTML so the important information is always available. Then you use CSS to improve the design and make the site look better. After that, JavaScript can be added for extra interactive features if the user’s browser supports it. This is important because it keeps websites accessible and easy to use. People with older browsers, weak internet, or disabled JavaScript can still access the main content without problems. At the same time, users with newer devices get the more advanced features and full experience. It also helps developers stay organized since the content, styling, and interactive parts are separated. I also read an article from MDN Web Docs about progressive enhancement, and it explained how this method helps create websites that are dependable, flexible, and able to work across different devices and situations.
